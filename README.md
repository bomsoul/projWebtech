INTRODUCTION
ระบบผู้ช่วยสำหรับอาจารย์ที่ปรึกษา เป็นระบบที่จะติดตามลักษณะของผู้เรียน เพื่อเป็นข้อมูลเบื้องต้นในการให้คำปรึกษาแก่ผู้เรียน โดยได้รับข้อมูลจากผู้สอนในรายวิชาต่าง ๆ และเป็นระบบผู้ช่วยสำหรับอาจารย์ผู้สอนที่ช่วยตรวจสอบเกรดของวิชาบังคับก่อน เกณฑ์ที่เคยใช้วัดผลการเรียนในภาคการศึกษาก่อนหน้า และช่วยวิเคราะห์เกณฑ์การวัดผลที่เหมาะสม
โปรเจคที่หนึ่งในรายวิชานี้ จะให้นิสิตออกแบบและพัฒนาเว็บไซต์ต้นแบบของระบบผู้ช่วยสำหรับอาจารย์ที่ปรึกษาและอาจารย์ผู้สอน โดยใช้เทคโนโลยีเว็บตามที่นิสิตได้ศึกษามา 
(เนื่องจากนิสิตถูกแบ่งกลุ่มให้ทำงานร่วมกับทีมงานที่ไม่คุ้นเคย ขอให้นิสิตเลือกเทคโนโลยีที่เหมาะสมกับกลุ่มเป็นสำคัญ)

ผู้ใช้ระบบ
ระบบนี้มีผู้ใช้ระบบ 3 กลุ่ม คือ 
1. ผู้ดูแลระบบ (Administrator) 
2. อาจารย์ผู้สอน และ/หรือ อาจารย์ที่ปรึกษา (Teacher) 
3. ผู้เรียน (Student)
Features
ระบบต้นแบบประกอบด้วย 5 features ดังนี้ 
System Users ผู้ใช้ระบบ
ผู้ดูแลระบบ เข้าสู่ระบบ (log in) ด้วย username/password ✩✩✩
ผู้ดูแลระบบ เปลี่ยนรหัสผ่าน (password) ของตนเองได้ ด้วยวิธีที่คำนึงด้านความปลอดภัย ✩✩
ผู้ดูแลระบบ เพิ่มอาจารย์เข้าไปในระบบผ่านหน้าเว็บ ✩✩✩
ผู้ดูแลระบบ เพิ่มผู้เรียนเข้าไปในระบบด้วยการ import CSV file ผ่านหน้าเว็บ ✩✩✩
ผู้เรียน เข้าสู่ระบบด้วย KGG (KU Goes Google) account (@ku.th) ✩✩✩
อาจารย์ เข้าสู่ระบบด้วย KGG account ✩✩✩
Student Behaviors พฤติกรรมผู้เรียน
อาจารย์ บันทึกพฤติกรรมของผู้เรียนได้ พฤติกรรมอาจจะเป็นการกระทำทั้งทางบวกและลบ อาการทางร่างกาย หรืออาการทางด้านอื่น เช่น ตาบอดสี โรคซึมเศร้า เข้าเรียนสายเป็นประจำ เป็นต้น ✩✩✩
อาจารย์ ตั้งค่าประเภทและระดับของพฤติกรรมผู้เรียนได้อย่างเหมาะสม ✩
อาจารย์ แก้ไขพฤติกรรมของผู้เรียนที่ตนเองเป็นผู้บันทึกได้ ✩✩✩
อาจารย์ ดูพฤติกรรมของผู้เรียนได้ ✩✩✩
Subjects การจัดการวิชาเรียน
ผู้ดูแลระบบ เพิ่มวิชาผ่านหน้าเว็บ และกำหนดว่าอาจารย์ท่านใดบ้างที่เป็นผู้สอนวิชาที่เพิ่ม ✩✩✩
ผู้ดูแลระบบ จัดการว่าวิชาใดบังคับก่อน (prerequisite) ได้ ซึ่งบางวิชามีวิชาบังคับก่อนมากกว่า 1 วิชา และเป็นไปได้ว่าต้องผ่านวิชาบังคับก่อนทั้งหมด หรือผ่านวิชาบังคับก่อนบางตัวก็ได้ ✩✩✩
อาจารย์ สร้างวิชาที่ต้องสอนในแต่ละภาคเรียนเพื่อจัดการคะแนนของภาคเรียนนั้นได้ ✩✩✩
อาจารย์ สร้างวิชาของตนเองที่ยังไม่มีในระบบได้ ✩
อาจารย์ เพิ่มผู้เรียนที่เรียนในวิชาได้ ด้วยการ import CSV file ผ่านหน้าเว็บ ✩✩✩
ผู้เรียน ขอสิทธิ์เข้าดูคะแนนในวิชาได้ด้วย code/qrcode กรณีที่ต้องการดูคะแนนของตนเอง ✩✩
อาจารย์ ลบการข้อมูลของผู้เรียนออกจากวิชาได้ผ่านหน้าเว็บ เสมือนว่าผู้เรียนไม่ได้เรียนในวิชานี้ ✩✩
Student Scores การจัดการคะแนนของผู้เรียน
อาจารย์ ดูคะแนนของผู้เรียนในวิชาตนเองได้ ✩✩✩
อาจารย์ เพิ่ม (รวมถึง import) คะแนนของผู้เรียนตามหัวข้อคะแนนในวิชาตนเองบนหน้าเว็บได้ ✩✩✩
อาจารย์ กำหนดคะแนนสูงสุดของแต่ละหัวข้อคะแนนได้ (แต่อาจให้คะแนนเกินได้) ✩✩
อาจารย์ แก้ไขคะแนน คะแนนสูงสุด หรือ แก้ไข/ลบ หัวข้อคะแนนได้ ✩
อาจารย์ กำหนดได้ว่าจะประกาศคะแนนของแต่ละหัวข้อเมื่อใด (ผู้เรียนจะทราบคะแนนเมื่ออาจารย์ประกาศคะแนนแล้วเท่านั้น) ✩✩
อาจารย์ export คะแนนในวิชาเป็น CSV file ได้ ✩✩✩
ผู้เรียน ดูคะแนนของตนเองในแต่ละวิชาบนหน้าเว็บได้ เมื่ออาจารย์ประกาศคะแนน โดยไม่เห็นคะแนนของผู้เรียนอื่น ✩✩✩
ผู้เรียน ได้รับข้อความแจ้งเตือนและคะแนนที่ได้ใน LINE เมื่อมีการประกาศคะแนนจากอาจารย์ ✩
Student Grade การจัดการผลคะแนน (เกรด) ของผู้เรียน
อาจารย์ ดูผลการเรียนของผู้เรียนในวิชาบังคับก่อนได้ กรณีที่วิชาของอาจารย์มีวิชาบังคับก่อน ✩✩✩
อาจารย์ import ผลการเรียนของผู้เรียนในวิชาตนเองได้ ✩✩
อาจารย์ กำหนดน้ำหนักของหัวข้อคะแนนเป็นร้อยละได้ 
เพื่อไปคำนวณคะแนนรายวิชาของผู้เรียน ✩✩
อาจารย์ กำหนดช่วงของคะแนนเพื่อใช้ในการวัดผลคะแนน (ตัดเกรด) ได้ โดยมีกราฟคะแนนของผู้เรียนประกอบการตัดสินใจ ✩✩
อาจารย์ กำหนดได้ว่าจะประกาศผลการเรียนของผู้เรียนเมื่อใด ✩✩
อาจารย์ export ผลการเรียนของผู้เรียนเป็น CSV file ได้ โดยกำหนดได้ว่าจะใช้ข้อมูลใดบ้างในการ export ✩✩✩
ผู้เรียน ดูผลการเรียนของตนเองในแต่ละวิชาบนหน้าเว็บได้ เมื่ออาจารย์ประกาศผลการเรียน โดยไม่เห็นผลการเรียนของผู้เรียนอื่น ✩✩✩
ผู้เรียน ได้รับข้อความแจ้งเตือนและผลการเรียนที่ได้ใน LINE เมื่อมีการประกาศผลการเรียนจากอาจารย์ ✩

Extra Features 
กรณีที่วิชาใดต้องการระบบเช็คชื่อ จะมีระบบเช็คชื่อเข้าชั้นเรียนที่ตรวจสอบได้ว่าผู้เรียนเข้าชั้นเรียนจริง กำหนดได้ว่าช่วงใดเป็นการเข้าชั้นเรียนทันเวลา ช่วงใดเข้าชั้นเรียนสาย โดยไม่รบกวนการเรียนการสอน และสรุปคะแนนเป็นหัวข้อคะแนนเข้าชั้นเรียนได้โดยอัตโนมัติ ✩
